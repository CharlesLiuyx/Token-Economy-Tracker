# ARR 估算方法论

> TL;DR：手工锚点（口径必填）→ 三种候选形态各自做对数空间加权最小二乘 → 滚动起点
> 回测选形态 → 冻结增速外推 + 随距离张开的置信带；页面常驻「估算」徽章 + 形态标签。
> 何时读我：改 scripts/derive/arr.py、加锚点、加候选形态、质疑大数字之前。
> 最后核对日期：2026-07-24（形态选择改造：加 Yipit 07-12 锚点后重写拟合）

## 数据

锚点在 `data/manual/arr_anchors.yml`，每条：
`company / date / value_usd / metric / source_url / note`。

**metric 口径必填**，混拟合会出鬼图：

- `run_rate_arr` —— 披露的年化运行率收入。**拟合只用这一种。**
- `projected_rev` —— 全年预测，仅展示为散点。
- `reported_rev` —— 事后实际收入，仅展示为散点。

## 模型

### 1. 候选形态（不预设，由数据选）

三种候选都在 **log 空间**做加权最小二乘（近期权重高，半衰期 `HALF_LIFE_DAYS`）——
ARR 跨三个数量级，误差是乘性的，log 空间才是对的度量。区别只在「增速怎么随时间走」：

| id | 形式 | 参数 | 说明 |
|---|---|---|---|
| `linear` | y = b·(t − t₀) | 2 | 恒定**绝对**增量（$/天不变），末端 MoM 被自己稀释 |
| `exponential` | ln y = a + b·t | 2 | 恒定**相对**增速（MoM 不变） |
| `log_quadratic` | ln y = a + b·t + c·t² | 3 | 相对增速线性漂移（在加速 / 在减速） |

`linear` 写成过 x 轴 t₀ 的形式是为了保证 y>0（log 空间可算），给定 t₀ 时 ln b 有闭式解，
故只需对 t₀ 做一维搜索。加新形态：在 `MODELS` 里加一项 + `arr_cards.j2` 的 `model_keys`
补 i18n key（漏了 build 会直接报错）+ 本表补一行。

### 2. 选形态：滚动起点回测

用信息准则（AIC 之类）要在加权似然的有效样本量上做文章，口径可争；这里直接测
**我们真正在乎的能力**：用前 i 个锚点拟合、预测第 i+1 个（它没见过），逐折推进，
取加权 RMSE（log 空间）最小者，同分取参数少的。折按目标锚点的近期权重加权——
2023 年预测得准不准，不该左右今天用哪个形态。

锚点刚够拟合（排不出折）时退化为「取参数最少的」：宁可欠拟合，也不让 3 个点
喂出一条 3 参数的完美曲线去外推。implied MoM 落在 `SANE_MOM_RANGE` 外的候选直接出局，
全出局才报错。

2026-07-24 实测（16/17 个 run-rate 锚点）：

| | linear | exponential | log_quadratic | 选中 |
|---|---|---|---|---|
| Anthropic 回测 RMSE | 1.0513 | 0.2776 | **0.2346** | `log_quadratic` |
| OpenAI 回测 RMSE | 0.5394 | **0.1100** | 0.1199 | `exponential` |

即：Anthropic 的增速本身在漂移（月增速从 2025 年的 ~11% 抬到 ~38%），OpenAI 是干净的
恒定指数（~12.5%/月）。**两家用了不同形态，这正是「由数据决定」的意思**；
`linear` 在两家都输两个数量级——ARR 曲线不是直线。

### 3. 外推：冻结增速

最后锚点之后，用**该点处的瞬时对数增速做直线延伸**，不把曲率外推出去。曲率是少数
锚点估出来的，t² 项会把误差放大，180 天外必然失真（实测 `log_quadratic` 若带曲率外推，
Anthropic 半年后比冻结增速再高 ~20%，且隐含 MoM 一路爬到 50%+，与近几段实际 MoM
正在回落的事实矛盾）。曲线在衔接点值与斜率都连续。

### 4. 置信带：预测区间，随距离张开

`σ²`（观测残差，按 Kish 有效样本量做自由度修正）+ `gᵀ·cov(β)·g`（参数不确定性），
取 95% 双侧。外推点的 `g` = 最后锚点处的设计向量 + h × 其对 t 的导数，所以**外推越远带越宽**
（Anthropic 实测 ×1.54 → ×2.23 @ +190 天）——恒定宽度的带是在撒谎。

**早期无支撑段直接不画**：近期加权 + 带曲率的形态往回推到几年前时，参数不确定性会炸开
（带宽 ×10 起）。那段曲线不是估算而是噪声，`_trim_unsupported` 把带宽超过 `MAX_BAND_RATIO`
的前段裁掉，只留锚点散点。稳定的指数形态不会被裁（OpenAI 仍从 2023-03 画起）。

### 5. 实时跳动

build 输出 `{arr_at_build, usd_per_hour, mom_implied}`（均取 build 日的瞬时增速），
前端每秒 `+= usd_per_hour / 3600`。**纯展示效果**，刷新回到 build 基准。

## 诚实性要求（PLAN §11 风险 2）

- 卡片常驻「估算」徽章 + **选中形态标签**（hover 说明形态是回测选出来的）+ 本文档锚链接。
- 走势图必须画锚点散点与置信带，让读者看到估算的地基。
- `data/derived/arr.json` 的 `fit.candidates` 留下全部候选的回测误差/σ/MoM 与落选原因，
  `fit.fitted_at_last_anchor` 与 `fit.last_anchor_value` 并列，便于一眼看出拟合有没有落后现实。
- 异常口径（metric 非法值 / 缺 source_url）在 derive 阶段直接拒绝。
