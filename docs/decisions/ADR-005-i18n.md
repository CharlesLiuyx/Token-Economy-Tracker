# ADR-005：i18n 用单文件运行时切换（zh/en），不做多份构建产物

> 状态：已决策 2026-07-13

## 背景

页面需支持中英双语。两条路线：
A）每语言构建一份 HTML（index.html + en/index.html），切换=跳链接；
B）单文件内联双语文案目录，运行时 JS 切换（同现有主题切换模式）。

## 决定

选 B。理由：单文件产物是本仓库硬约束的核心身份（离线可开、单 URL、
GitHub Pages 零改动）；双语目录仅 ~10KB，路线 A 则整站体积翻倍且引入
多产物部署面。机制完全复用已有模式，不新增框架：

- **文案唯一出处** `site/template/i18n.yml`（key -> {zh, en}），加载时校验缺语言即失败。
- **静态 DOM**：模板 `{{ t(key) }}` 渲染构建期默认语言（zh，无 JS 时兜底），
  同元素标 `data-i18n`（textContent）/ `data-i18n-html`（受信 HTML）/
  `data-i18n-title` / `data-i18n-aria-label`；参数化文案用 `data-i18n-args`
  （JSON，`{name}` 占位）。app.js 切换时全量重渲染。
- **图表 spec**：build.py `L(key)` 生成 `"__i18n:<key>"` 占位符（可嵌在
  `f"{company} {L(...)}"` 长串中），app.js 渲染时正则解析——与既有
  `__surface` 占位符同一套解析管线；语言切换像主题切换一样销毁重建图表。
- **偏好**：localStorage `lang` 覆盖 `navigator.language`（zh* → zh，否则 en），
  head 内联脚本首帧前设 `data-lang`；`<html lang>` 随切换更新。

## 后果

+ 单 URL/单产物不变；新增语言 = i18n.yml 每条加一列 + LANGS 加一项。
+ tests/test_i18n.py 三道闸防腐化：zh/en 齐全、`{args}` 双语一致、
  引用与目录双向对齐（防死 key / 漏 key）；模板与 spec 不允许硬编码可见文案。
− 数据内容不翻译（新闻标题、引语、站点/模型名——保持原文）；
  构建产物默认渲染 zh，en 用户首帧后由 JS 换文案（同步脚本，实测无闪烁）。
