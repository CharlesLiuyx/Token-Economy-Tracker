Methodology — 每个面板测的是什么、不是什么

  • 01 ARR：披露锚点（run-rate 口径）对数空间加权最小二乘拟合 + 外推。曲线形态由数据决定： linear（恒定绝对增量）/ exponential（恒定月增速）/ log_quadratic（月增速在漂移）三种候选各自拟合， 用滚动起点回测挑预测下一个锚点最准的那个，卡片上标出选中形态。外推区沿用最后锚点处的瞬时增速直线延伸， 不把曲率外推出去。大数字是估算的演出效果（每秒按拟合小时增量累加），非实时财务数据； 置信带（随外推距离张开）与锚点散点展示估算地基。锚点混入全年预测/实际收入会被拒绝（口径字段强制）。
  • 02 OpenRouter：聚合路由平台切片，不含 Anthropic/OpenAI 直连 API 主力流量。 「滚动 7 天」为端点原生口径；日频序列由本站每日快照累积。
  • 02 Vercel Gateway：仅 Gateway 客户流量的份额百分比，无绝对量；模型名为显示名。
  • 02 SDK 下载：npm/PyPI 官方 API；下载量是开发者采用度 proxy（CI 重复安装会灌水）， 不等于 API 调用量。
  • 03 GPU 价格：Vast.ai 市场 on-demand 报价分位数（含个人/小机房供给的低价切片）， 系统性低于超算云牌价，不可与 OCPI 等跨供应商指数直接比较。
  • 04 数据中心：Epoch AI Frontier Data Centers 数据集（CC-BY，约每周更新）， 为样本内站点，非全行业普查。新闻流为 Google News RSS 白名单媒体，仅标题/链接。
